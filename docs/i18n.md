# Translations

Translations are plain JSON files in `ui/i18n/`, one per language. No build step, no dependency. `en.json` is the source and the fallback for any missing key.

## Add a language

1. Add it to `LANGUAGES` in `ui/js/i18n.js`:

   ```js
   { code: 'it', name: 'Italiano', dir: 'ltr' },
   ```

   `code` is the filename (`ui/i18n/it.json`), `name` is the language's own name
   shown in the selector, `dir` is `ltr` or `rtl` (RTL flips the whole layout).

2. Copy `en.json` to `ui/i18n/<code>.json` and translate the values.

The language then appears under Settings, General, Language. Anything left untranslated falls back to English, so partial catalogs are fine.

## Translator notes

- Translate values, not keys; keep the JSON structure identical to `en.json`.
- Keep placeholders (`{name}`, `{n}`, ...) and any HTML tags (`<strong>`, `<code>`) intact.
- Proper nouns and technical tokens (Unsplash, Docker, URLs) can stay in English.

Widget can opt in by importing `t` from `/js/i18n.js`, but it's not required.
